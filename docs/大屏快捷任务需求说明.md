# 大屏快捷任务需求说明

文档状态：评审稿  
编制日期：2026-07-23

关联文档：

- [大屏快捷任务技术方案](大屏快捷任务技术方案.md)
- [大屏快捷任务接口说明](大屏快捷任务接口说明.md)

## 1. 目标

大屏为机器人提供以下快捷任务入口：

- 返回驻点。
- 自主回充。
- 驶离充电桩。

这些操作由管理端提前编排，大屏只负责查询、触发和展示执行结果。

## 2. 业务名称

| 页面名称 | 任务编码 | 含义 |
|---|---|---|
| 返回驻点 | `RETURN_HOME` | 返回指定驻点，不一定是充电桩 |
| 自主回充 | `DOCK` | 返回充电桩并完成对接 |
| 驶离充电桩 | `UNDOCK` | 从已对接的充电桩安全驶离 |

不使用“一键返航”“退出充电桩”等含义不够明确的名称。

## 3. 业务边界

本需求包含：

- 查询机器人可执行的快捷任务。
- 触发管理端已编排的任务。
- 展示任务状态、进度和失败原因。
- 处理任务与人工控制的冲突。
- 处理单地图、多地图和充电桩差异。

本需求不包含：

- 管理端任务编排器的内部实现。
- ROS2、厂商 SDK 和充电桩底层接口实现。
- 地图制作、SLAM 和路径规划算法。

## 4. 页面展示规则

1. 管理端没有为机器人配置某项任务时，不显示对应按钮。
2. 任务已配置但当前不能执行时，显示禁用按钮和原因。
3. 任务可执行时，按钮正常显示。
4. 自主移动任务点击后需要二次确认。
5. 任务提交后，页面显示等待、执行中、成功或失败。
6. HTTP 请求成功不等于任务成功，最终结果以任务实例状态为准。

## 5. 返航规则

### 5.1 一期：返回当前地图驻点

一期按机器人当前地图选择返航任务：

```text
机器人当前地图
  -> 查询该地图配置的返回驻点任务
  -> 创建任务实例
  -> 管理端下发任务
  -> 机器人执行
```

要求机器人状态提供：

- `currentMapId`：当前地图。
- `localizationStatus`：当前定位状态。

以下情况不能返航：

- 当前地图未知。
- 定位不可用。
- 当前地图没有配置驻点。
- 机器人不支持导航。
- 急停生效。
- 存在冲突任务或本体人工控制。

### 5.2 二期：跨地图返航

跨地图返回默认驻点需要具备：

- 地图连接关系。
- 电梯或门禁联动。
- 地图切换。
- 目标地图重新定位。
- 跨地图任务进度上报。

一期未实现跨地图能力时，不显示或禁用“返回默认驻点”，不得把其他地图坐标直接发给机器人。

## 6. 充电桩规则

### 6.1 自主回充

只有满足以下条件时才可执行：

- 机器人绑定了充电桩。
- 管理端配置了回充任务。
- 机器人和充电桩状态允许执行。

### 6.2 驶离充电桩

只有满足以下条件时才可执行：

- 机器人绑定了充电桩。
- 管理端配置了驶离任务。
- 机器人上报 `dockingStatus=DOCKED`。

驶离任务应由管理端编排停止充电、解锁、驶离和脱离确认等步骤。

## 7. 人工控制冲突

- 本体正在被人工控制时，不允许启动冲突的快捷任务。
- 一期返回提示，要求用户先释放人工控制权。
- 任务执行中发起人工接管时，应先暂停或取消任务，再进入手动模式。
- 任务启动时由任务链路完成模式切换，前端不单独切换控制模式。

## 9. 一期建议范围

| 功能 | 建议 |
|---|---|
| 当前地图返回驻点 | 一期 |
| 驶离充电桩 | 一期 |
| 自主回充 | 根据现有任务能力确认 |
| 任务状态和失败原因 | 一期 |
| 人工控制冲突 | 一期 |
| 任务取消 | 二期 |
| 跨地图返航 | 二期 |
| 最近驻点自动选择 | 二期 |
| 人工接管后恢复原任务 | 二期 |

## 10. 验收标准

1. 不支持某项任务的机器人不会显示对应按钮。
2. 暂时不能执行的任务显示禁用原因。
3. 点击任务后生成唯一 `taskExecutionId`。
4. 页面能区分任务已创建、执行中、成功和失败。
5. 机器人当前地图变化后，不会执行旧地图返航任务。
6. 没有驻点、没有定位或没有充电桩时不会错误执行。
7. 人工控制占用时不能启动冲突任务。
8. 快捷任务不会进入本体直接控制 topic。
9. 任务失败时页面显示明确原因。
10. 任务操作可以查询和审计。

## 11. 评审需确认

1. 是否同时保留“返回驻点”和“自主回充”。
2. 一期是否确定只做当前地图返航。
3. 自主回充是否纳入一期。
4. 驶离任务是否允许取消。
5. 任务执行中人工接管采用暂停还是取消。
6. 任务结束后机器人恢复哪种控制模式。
7. 管理端是否已有人工立即执行任务的接口。
8. 当前地图和停靠状态由哪个接口提供。
